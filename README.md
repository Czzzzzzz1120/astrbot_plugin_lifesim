# astrbot_plugin_lifesim - 模拟人生小游戏插件

一个基于 AstrBot 的模拟人生文字游戏插件。支持私聊和群聊（含 NapCat QQ 端），玩家可以创建角色、选择世界观、分配属性点，经历从出生到死亡的完整人生模拟。大模型驱动叙事，每个关键年龄节点都有随机事件选择，影响角色属性和人生走向。

## 功能特性

- **大模型驱动叙事** — 支持从 AstrBot 已添加的模型中直接选取，或手动配置 OpenAI 兼容 API
- **8 个预设世界观 + 任意自定义** — 武侠江湖、都市白领、仙侠修真、末日生存、古代宫廷、奇幻冒险、赛博朋克、海岛求生，也可输入任意世界观名称
- **AI 实时叙事** — 无固定生命阶段，所有日常事件和重大事件均由 AI 根据世界观、角色年龄、过往经历实时生成，剧情连贯
- **5 属性系统** — 体质、智力、颜值、快乐、家境，每项初始 5 点，自由分配 10 点，无上限
- **真实属性变化** — 代码预决定属性变化 → AI 围绕变化写叙事 → 【体质+1/智力-1】真实有效，下限 1、无上限
- **属性→叙事联动** — 代码先随机属性变化，AI 围绕变化写一句 30 字内叙事，叙事与【】永远一致
- **角色创建系统** — 自定义姓名、性别（男/女/其他）、AI 随机生成出身背景（8种类型+AI描述）、天赋5选3
- **随机天赋系统** — 30 种天赋按稀有度加权随机，创建角色时从 5 个中选 3 个
- **自由加点** — 体质、智力、颜值、快乐、家境五项属性，10 点自由分配，支持快速分配和随机分配
- **自动推进** — 非大事件年份自动推进 5 年并批量展示剧情，无需逐年输入"人生继续"（每年叙事更丰富，80-120字）
- **随机重要事件** — 每十年至少 2 个重大事件，AI 动态生成具体场景叙事 + 3 个选项 + 1 个自定义，日常事件与重大事件之间保持因果连贯
- **风险评估系统** — 重要事件选项标注 🟢低/🟡中/🔴高 风险等级，高风险选择可能导致属性惩罚或严重后果
- **AI 智能死亡判定** — 由 AI 根据角色属性、年龄、事件背景综合判断是否死亡，死亡原因由 AI 根据情境生成
- **超时保护** — 5 分钟无操作自动结束游戏，防止游戏卡住
- **随时重置** — `人生重置` 在任何阶段都可使用，随时重新开始
- **人生评价** — 角色死亡后展示「人生重开模拟器」风格结算面板（属性进度条 + 等级评语 + 幽默总评），AI 生成人生总结
- **群聊适配** — 群聊中每位玩家独立游戏，自动 @回复，个人剧情带 📖 标记区分
- **群内排行榜** — 群聊中可查看所有玩家的游戏进度排行
- **AstrBot 日志适配** — 游戏关键操作输出结构化日志，方便运维排查

## 安装

### 方式一：插件市场

在 AstrBot WebUI → 插件市场中搜索 `astrbot_plugin_lifesim`，点击安装。

### 方式二：手动安装

```bash
cd AstrBot/data/plugins
git clone https://github.com/Czzzzzzz1120/astrbot_plugin_lifesim.git
pip install openai
```

安装后在 AstrBot WebUI 中重载插件。

## 配置

在 AstrBot WebUI → 插件管理 → 设置中配置：

| 配置项 | 说明 |
|--------|------|
| **对话模型** | 从 AstrBot 已添加的模型中选择（推荐，优先级最高） |
| 手动配置 - API Key | 未选择上方模型时生效，输入 OpenAI 兼容的 API Key |
| 手动配置 - API 地址 | 默认 `https://api.openai.com/v1`，可改为 DeepSeek 等 |
| 手动配置 - 模型名称 | 如 `gpt-4o`、`deepseek-chat` 等 |

> 不配置模型也可游玩，但叙事将使用预设文本，不会由 AI 生成。

## 指令列表

所有指令均以 `人生` 为前缀，避免与其他插件冲突。每个指令都有 `rl` 前缀的英文别名，方便快速输入。

| 指令 | 别名 | 说明 |
|------|------|------|

| `人生选择 <选项>` | `rlchoose` `rlxz` | 做出选择 / 输入名字 / 自定义行动 |

| `人生加点 6 9 9 6` | — | 快速分配（力量 智力 魅力 运气） |
| `人生加点 随机` | — | 随机分配全部剩余点数 |
| `人生继续` | `rlnext` `rljx` | 推进人生（自动推进 5 年，遇事件停下，每年 80-120 字） |
| `人生天赋` | `rltalent` `rltf` | 查看当前天赋 |
| `人生属性` | `rlattr` `rlsx` | 查看当前属性面板 |
| `人生状态` | `rlstatus` `rlzt` | 查看完整游戏状态 |

## 重要事件系统

人生模拟过程中会随机触发重要事件，**每个十年至少保证 2 个事件**，事件从 1 岁一直分布到预计寿命之后，确保人生全程都有抉择体验。每个事件都有适合的年龄范围，确保事件与人生阶段匹配。

事件类型包括：天赋觉醒、童年危机、贵人相助、少年磨难、命运转折、情感风暴、事业起步、重大抉择、危机降临、中年觉醒、命运再次考验、人生巅峰、传承之责、暮年危机、人生终章等 15 种。

### AI 动态生成场景与选项

每个重要事件触发时，AI 会根据事件主题、角色当前属性、出身、世界观和近期经历，**一次性生成具体的场景叙事（80-120字）和 3 个应对选项**，确保上下文连贯、剧情有代入感。每个选项都有风险等级和属性加成提示：

- 🟢 **低风险** — 稳定收益（+2属性点），描述角色的稳妥行动
- 🟡 **中风险** — 有一定变数（+2-3属性点），20% 概率获得额外奖励
- 🔴 **高风险** — 高风险高回报（+3-4属性点），40% 概率遭受属性惩罚，可能导致严重后果

重要事件选项格式：
```
⚡ 命运转折 ·25岁
收到一封密信，远房叔父留下巨额遗产，需穿越匪患横行的黑风岭。
1.[冒死穿越黑风岭]🔴家境+3
2.[雇镖师护卫同行]🟡智力+2 颜值+1
3.[放弃遗产留在小镇]🟢家境+2
4.自定义
人生选择 <1/2/3> →
```

### AI 智能死亡判定

游戏不再使用硬编码的属性死亡逻辑，而是由 AI 根据角色的完整处境智能判断是否死亡：

- **综合考量** — AI 会评估角色的属性值、年龄、近期事件、选择风险等多重因素
- **属性影响** — 属性过低（≤3）会增加死亡概率，属性极低（≤1）时死亡概率极高
- **剧情合理** — 死亡原因由 AI 根据具体情境生成，而非预设的固定文案
- **高风险后果** — 高风险选择可能导致 AI 判定死亡，具体取决于角色当前状态

死亡判定在每次推进人生和重大事件选择后触发，确保剧情的合理性和紧张感。

## 人生评价

角色死亡后，系统会生成模仿「人生重开模拟器」风格的结算面板：

### 评分计算

| 评分项 | 计算方式 |
|--------|---------|
| 属性分 | 五项属性总和 |
| 天赋分 | 天赋数量 × 5 |
| 抉择分 | 重要抉择次数 × 3 |
| 年龄分 | 存活年数 × 0.5 |

总分对应等级：SSS(≥180) → SS(≥145) → S(≥110) → A(≥85) → B(≥60) → C(≥40) → D(≥25) → F(<25)

## 群聊功能

### 基本特性
- 群聊中每位玩家拥有独立的游戏进程，互不干扰
- 所有回复自动 @发送者，方便多人同时游戏
- 个人剧情带 📖 角色名标记，区分不同玩家的故事
- 支持 `人生排行` 命令查看群内所有玩家的游戏进度
- 游戏过程中自动阻止 LLM 聊天流，避免消息串流干扰

### NapCat QQ 端适配
本插件已针对 NapCat（OneBot V11 协议）进行适配：
- 消息自动分段，避免触发 QQ 的合并转发（超过 1500 字符会自动转为合并转发）
- 死亡总结分段发送：死亡事件 → 评分评价 → 关键抉择记录
- 长文本排行榜自动分段，不会因人数过多导致消息异常
- 正确处理 NapCat 中 sender name 为 "N/A" 的情况
- At 组件格式兼容，QQ 群聊中正常显示 @效果

### 使用方式
在 QQ 群中使用时，发送指令即可（无需 @机器人）：
```
人生开启 仙侠修真        ← 使用预设世界观
人生开启 蒸汽朋克维多利亚  ← 任意自定义世界观
人生选择 男
人生选择 张三
人生选择 1 3 5      ← 天赋5选3
人生加点 3 3 2 2
人生继续    ← 自动推进5年，遇大事件停下
人生选择 2  ← 选择事件选项（含风险评估）
人生继续    ← 继续自动推进
```

## 输出样式

插件采用简洁有趣的输出格式，代码预决定属性变化，AI 围绕变化写一句叙事（不超过 30 字），【】由代码独立拼接，永远一致：

```
 0岁：你出生在一个小康家庭，父母看着你皱巴巴的小脸，露出了欣慰的笑容。
 1岁：你学会了走路，虽然摔了无数次，但体质似乎变好了一点。【体质+1】
 2岁：你沉迷于看蚂蚁搬家，展现了惊人的专注力。【智力+1】
 3岁：妈妈教你背唐诗，你当着亲戚的面背错了三首，把"鹅鹅鹅"背成"饿饿饿"，逗得大家哈哈大笑。【颜值+1】
 6岁：你上了小学，因为零花钱多，成为了班里的零食大王，人缘极好。【快乐+3】
 8岁：你迷上了打游戏机，视力开始下降，但反应速度变快了。【体质-1，智力+1】
 10岁：天天在街头疯跑爬树，摔得浑身是泥但越来越皮实。【体质+1，智力-1】
```

每年叙事包含具体小事和内心感受，【】内属性变化 100% 真实有效且与叙事一致。重要事件会显示风险等级供选择。

## 游戏管理

### 随时重置
`人生重置` 命令在游戏的任何阶段都可以使用，包括角色创建中途、人生模拟进行中、角色死亡后等所有状态。

### 超时自动结束
如果 5 分钟内没有任何游戏操作，游戏会自动结束并清除状态。此时直接输入 `人生开启 <世界观>` 即可开始新游戏，无需手动重置。

### 自动推进
输入 `人生继续` 后，非大事件年份会自动推进 5 年并批量展示剧情（每年叙事 30-50 字），无需逐年操作。遇到重要事件时会自动停下，显示风险评估和选项供玩家选择。

### 重新开始
- 游戏进行中：输入 `人生重置` 清除当前进度，然后 `人生开启 <世界观>` 开始新游戏
- 角色死亡后：直接输入 `人生开启 <世界观>` 即可开始新的人生
- 超时后：直接输入 `人生开启 <世界观>` 自动清理并开始新游戏

### AstrBot 日志
插件在 AstrBot 日志中输出以下关键信息：
- 游戏开始/重置/超时清理
- 重要事件触发和玩家选择（含风险等级）
- 角色死亡原因、评分、等级
- 自由加点完成时的角色属性
- LLM 调用失败等错误信息

日志格式示例：
```
[pid] New game started: world=仙侠修真
[pid] Character created: 张三, attrs={'strength': 8, ...}, events=24
[pid] Born: age=1, stage=playing
[pid] Auto-advance start: age=1, stage=playing
[pid] Auto-advance: batch 5 years, ended at age=6
[pid] Important event at age 10: 天赋觉醒
[pid] Important event choice: 大胆展示 (risk=中) at age 10
[pid] Died at age 78: 衰老 (chance=0.350)
[pid] Game over: 张三 died at 78 (衰老去世), grade=S, score=95
```

## AI 叙事系统

游戏没有固定的"生命阶段"划分，所有日常事件均由 AI 根据以下信息实时生成：

- **世界观** — 武侠江湖中可能遇到江湖纷争，仙侠修真中可能遇到灵脉探秘
- **角色年龄** — 幼年时期的启蒙、青壮年的冒险、暮年的感悟
- **过往经历** — 日常事件与重大事件之间保持因果连贯，形成完整的故事线
- **角色属性** — 属性状态直接影响叙事内容和事件走向

### 属性→叙事联动

代码先随机决定属性变化，再将变化告知 AI，AI 围绕变化写一句叙事：

- **有属性变化时**：`{姓名}{年龄}岁。今年{体质+1，智力-1}。围绕这个变化写一句30字内叙事，不写【】`
- **无属性变化时**：`写一句{姓名}{年龄}岁时的生活片段，不超过30字`

属性变化由代码预决定，AI 不负责决定属性，只负责写叙事，【】由代码独立拼接。叙事与属性变化永远一致。

### 事件连贯性

- 日常事件会参考近期经历和最近一次重大抉择，确保叙事不脱节
- 重大事件会基于日常事件积累的背景，自然地从剧情中引出
- AI 会根据世界观生成符合该世界设定的事件，而非使用预设事件池

## AstrBot 行为模块适配

本插件已完整适配 AstrBot 的行为模块（Star 框架）：

- **指令隔离** — 所有游戏指令均设置 `should_call_llm(False)`，插件处理完指令后不会触发主 LLM 对话，避免消息干扰和 token 浪费
- **指令命名空间** — 所有指令以 `人生` 为前缀，英文别名以 `rl` 为前缀，与 AstrBot 其他插件完全隔离，不会产生指令冲突
- **群聊 @回复** — 群聊中所有游戏回复自动 @发送者，使用 `event.make_result().at()` 构建消息
- **超时保护** — 5 分钟无操作自动清理游戏状态，防止内存泄漏
- **线程安全** — 使用 `asyncio.Lock` 保护游戏状态，确保多人同时游戏时不会产生数据竞争
